# Исправление ошибки "useEffect is not defined" в TimeSeriesAnalyzer

## Проблема

При загрузке страницы "Анализ данных" возникала ошибка:

```
Uncaught ReferenceError: useEffect is not defined
    at TimeSeriesAnalyzer (TimeSeriesAnalyzer.tsx:80:3)
```

## Причина

В файле `src/components/TimeSeriesAnalyzer.tsx` использовался хук `useEffect`, но он не был импортирован из React.

## Решение

Добавлен импорт `useEffect` в строку импорта React:

```tsx
// Было
import React, { useState, useRef, useCallback, useMemo } from 'react';

// Стало
import React, { useState, useRef, useCallback, useMemo, useEffect } from 'react';
```

## Результат

После исправления:

- ✅ **Ошибка устранена** - `useEffect` теперь доступен
- ✅ **Страница "Анализ данных" загружается корректно**
- ✅ **Инициализация полей договора работает** как ожидается
- ✅ **Реквизиты договора отображаются** с данными из проекта

## Тестирование

Для проверки исправления:

1. Откройте страницу "Анализ данных"
2. Убедитесь, что страница загружается без ошибок
3. Проверьте, что в блоке "Реквизиты договора" отображаются данные:
   - № договора: "ВР-15-25"
   - Дата договора: "2025-10-01"

## Дополнительная информация

Эта ошибка возникла при добавлении функциональности автоматической инициализации полей договора данными из проекта. Хук `useEffect` необходим для отслеживания изменений в пропе `project` и автоматического заполнения полей `contractFields`.
