# Оптимизация Git команд

## Проблема
Команда `git diff HEAD` может зависать на больших файлах или при большом количестве изменений.

## Оптимизированные альтернативы

### 1. Проверка статуса (самый быстрый)
```bash
git status
git status --short
git status --porcelain
```

### 2. Только имена измененных файлов
```bash
git diff --name-only HEAD
git diff --name-status HEAD
```

### 3. Статистика изменений (без содержимого)
```bash
git diff --stat HEAD
git diff --stat --max-count=20 HEAD  # ограничение количества файлов
```

### 4. Конкретный файл
```bash
git diff HEAD -- path/to/file.tsx
```

### 5. Ограничение вывода
```bash
# PowerShell
git diff HEAD | Select-Object -First 50
git diff HEAD -- path/to/file.tsx | Select-Object -First 100

# Linux/Mac
git diff HEAD | head -50
git diff HEAD -- path/to/file.tsx | head -100
```

### 6. Проверка конкретных паттернов
```bash
# Только строки с определенным паттерном
git diff HEAD | Select-String -Pattern "template|шаблон"
git diff HEAD -- path/to/file.tsx | Select-String -Pattern "template"
```

## Рекомендуемый порядок проверки

1. **Сначала:** `git status` - быстрая проверка изменений
2. **Затем:** `git diff --name-only` - список измененных файлов
3. **При необходимости:** `git diff --stat` - статистика
4. **Для конкретного файла:** `git diff HEAD -- path/to/file.tsx`

## Избегать

❌ `git diff HEAD` без ограничений на больших проектах
❌ `git diff` без указания конкретных файлов при большом количестве изменений
